| [Мультисервисное приложение: удержание и рост повторных заказов](./multiservice_retention/) | **Что было в задаче:** провести EDA и **самостоятельно найти продуктовую проблему**, сформулировать гипотезы и выбрать приоритетную.<br><br>**Проблема, которую я нашла:** половина пользователей **не пользуются сервисами вообще**; среди тех, кто попробовал — высокий отток после 1-го заказа, особенно в «Афише» (**58%**) и «Продуктах» (**43%**) vs «Топливо» (**17%**).<br>**Анализ:** сегментация пользователей по комбинациям сервисов (в т.ч. крупный сегмент «только Топливо»), сравнение **rolling retention 30/60/90 дней** по сервисам, поиск вертикалей/сегментов с просадкой повторных заказов.<br>**Гипотезы роста:**<br>1) **Баннер + мини-игра после покупки** (показывать только после успешной транзакции) → увеличить Repeat30/Second-order через «момент радости».<br>2) **Купон на “Продукты” после заправки** (ограничение 24 часа + автоприменение) → ускорить adoption «Продуктов» и рост доли 2-го заказа в сегменте Fuel.<br>**Приоритизация:** ICE → выбрана гипотеза №1 (**448 > 210**).<br>**Артефакты:** [PDF](./multiservice_retention/Multu.pdf) · [Jupyter Notebook](./multiservice_retention/Multu.ipynb) | **Python** (pandas, numpy, scipy) · matplotlib/seaborn · Jupyter · ICE |
| [Superset: дашборды и продуктовая диагностика](./superset_dashboard/) | **Что делала:** 2 кейса в Superset: (1) retention рекламной кампании + расследование падения DAU (сегментация, проверка гипотез причин, план мониторинга), (2) аудитория по неделям (new/retained/gone) для контроля изменений.<br>**Результат:** дашборды + набор метрик/срезов для регулярного мониторинга. | **Apache Superset** · **SQL** · продуктовые метрики (DAU/Retention) |
